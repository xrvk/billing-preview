Hi there! We're thrilled that you'd like to contribute to this project. Your help is essential for keeping it great.

Contributions to this project are [released](https://docs.github.com/en/site-policy/github-terms/github-terms-of-service#6-contributions-under-repository-license) to the public under the [project's open source license](LICENSE).

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.

## Data privacy

Do not submit real customer data, billing reports, screenshots with billing information, or other sensitive information in issues, pull requests, discussions, fixtures, examples, snapshots, or documentation.

Use obviously synthetic values such as `test-user`, `mona`, `octocat`, `hubot`, `example-org`, and `Cost Center A`.

## Prerequisites

- [Node.js](https://nodejs.org/) 20 or later
- npm

## Submitting a pull request

1. [Fork][fork] and clone the repository
2. Install the dependencies: `npm install`
3. Make sure linting passes: `npm run lint`
4. Make sure tests pass: `npm run test`
5. Create a new branch: `git checkout -b my-branch-name`
6. Make your change and make sure linting and tests still pass
7. Push to your fork and [submit a pull request][pr]
8. Pat yourself on the back and wait for your pull request to be reviewed and merged.

Here are a few things you can do that will increase the likelihood of your pull request being accepted:

- Keep your change as focused as possible. If there are multiple changes you would like to make that are not dependent upon each other, consider submitting them as separate pull requests.
- Write a [good commit message](https://tbaggery.com/2008/04/19/a-note-about-git-commit-messages.html).
- Update tests and documentation when changing parsing, products, SKUs, included credits, or aggregation logic.
